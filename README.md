# Джулюница · кирилица без грешки

> Платформа за **двойки шрифтове** с фокус върху българска кирилица — едно място, по-малко „руски шрифт за всичко“ и повече смислен дизайн.

---

## Защо това е ESG (и не е stretch)

- **Social / inclusion** — езикът и писмеността са част от това дали цифровите продукти са достъпни и уважават аудиторията
- **Governance / качество** — един каталог с правила и подбор намалява хаоса: по-малко „кой каквото намери“, повече последователност за брандове и екипи.
- **По-малко ресурс, изхабен на грешки** — когато дизайнерите и dev-овете не губят време в random шрифтове и ъпдейти заради грешен glyph set, това е **по-ефективна работа** = по-малко ненужен труд (и косвено по-малко digital waste от преправяния).

Тоест: **ESG при нас ≠ само климат**, а **как правим цифровия свят по-честен, по-ясен и по-малко разхищителен** — през нещо „скучно“ като шрифтовете.

---

## Какво има в проекта rn

Фронтенд-only, без backend (за момента). Работи като отвориш `home.html` локално или с лек static server.

### Ядро на каталога

- **`home.html`** — начална страница, филтри, сортиране, intro за продукта, линкове към contact/about/privacy.
- **`style.css`** — layout, grid, карти, чат widget, responsive.
- **`main.js`** — категории (всички / серифни / безсерифни / декоративни), филтри (комбинация, стил, употреба, кирилица), сортиране, pagination „Още“, рендер на картите, **контекст за AI** (`getAssistantContext` — какво гледаш и къде кликаш).
- **`fonts_list.js`** — тегли **Google Fonts API**, филтрира семейства с **cyrillic**, генерира двойки (serif+sans, sans+sans, и т.н.), merge с Adobe списъка.
- **`adobe_catalog.js`** — статични примерни двойки с **Adobe Fonts** (чрез Typekit kit id) + Google за тяло — за да се вижда идеята „всички източници на едно място“. Kit ID се попълва от теб в файла.

### Картите с шрифтове

- Горе: **фиксиран образец** (заглавие + къс параграф от данните).
- Под името на двойката: тагове и рейтинг.
- Зона **„Свой образец с кирилица“** — полета за **свой текст**; жив преглед с шрифта за заглавие и за текст (реагира на слайдера за размер).
- Бутон **„Детайли“** — placeholder за бъдещо разширение.

### Чат / асистент

- **`chatbot.js`** — UI за съобщения; може да вика **`CHATBOT_BACKEND_URL`** с `{ message, context }`. Без backend има fallback отговори на български + ползва контекста от каталога (филтри, видими двойки, кликнати карти).

### Други файлове

- **`contact.html`**, **`about.html`**, **`privacy.html`** — статични страници.

## Технологии

- **HTML5** – Структурата (header, main, sidebar, product grid, footer).
- **CSS3** — оформление и визия: Flexbox, CSS Grid, custom properties (variables), media queries за responsive. Без CSS framework; стиловете са custom в style.css.

- **JavaScript (vanilla)** — цялата логика: смяна на категории, филтри, сортиране, рендер на картите с двойки шрифтове, „Още“, полета за собствен образец с кирилица и чат асистент. Без front-end framework.

- **Данни за шрифтове** — динамичен каталог от Google Fonts Web Fonts API в fonts_list.js (филтър по cyrillic), плюс статичен масив в adobe_catalog.js за примерни Adobe + Google двойки; зареждат се с <script> преди main.js.

- **Adobe Fonts** — уеб шрифтове чрез Typekit (use.typekit.net), когато е зададен ADOBE_FONTS_KIT_ID в adobe_catalog.js.

---

## Как да го пуснеш

1. Клонирай репото.
2. В `fonts_list.js` си има API key за Google (което в бъдеще ще бъде ограничено, но с цел демонстрация остава засега тук) 
3. В `adobe_catalog.js` сложи **`ADOBE_FONTS_KIT_ID`**, ако искаш да се зареждат Adobe страните на шрифта.
4. Отвори `home.html` или: `python -m http.server` в папката и влез на `localhost`.
